# 디지털업무개선 연구대회 출품 기획안

## 프로그램명(가칭): 전임자의 서랍 — 교육기관 내부망 문서 통합검색·업무 지식베이스

---

## 1. 기획 배경

### 1.1 문제 인식: "그 문서, 분명히 있는데 못 찾는다"

교육청·직속기관·학교를 가리지 않고 공통으로 겪는 고충은 **과거 산출물을 찾지 못해 같은 문서를 다시 만드는 일**이다.

- 부서 공유폴더(NAS)에는 수년치 한글(hwp)·PDF·엑셀 파일이 쌓여 있지만, 탐색기는 **파일명으로만** 검색된다. 파일명은 `계획(최종)_수정2.hwp` 같은 식이라 사실상 검색 불가
- 업무관리시스템(K-에듀파인)의 문서 검색은 제목·기안자 중심이라 **본문 내용으로는 찾을 수 없고**, 첨부파일 속 내용은 더더욱 불가
- 인사이동 주기가 짧아(평균 1~2년) 담당자가 바뀌면 전임자의 폴더 구조를 해독하는 데만 수일이 걸리고, 결국 **작년 계획서·정산 서식·유사 공문을 처음부터 다시 작성**
- 인수인계서는 형식적 1~2장에 그치는 경우가 많아 실질 지식은 폴더 속에 묻힘

### 1.2 왜 기존 도구로 안 되는가

| 도구 | 한계 |
|------|------|
| 윈도우 탐색기 검색 | hwp 본문 미지원, 느림, 파일명 위주 |
| 업무관리시스템 검색 | 제목·기안자 중심, 첨부 본문 검색 불가, 부서 공유폴더 미포함 |
| 구글 드라이브·노션 등 클라우드 | 행정자료의 외부 민간 클라우드 저장 불가(보안 지침) |
| 상용 기업검색 솔루션 | 고가 라이선스, 학교·과 단위 도입 불가능 |
| 생성형 AI 서비스 | 내부 문서를 외부 API로 전송해야 함 → 보안상 원천 차단 |

**공백 지점: 외부망이 차단된 교육기관 환경에서, 한글(hwp/hwpx) 본문까지 읽어내는 무료 검색 도구가 없다.** 이 공백을 메우는 것이 본 연구의 차별성이다.

### 1.3 일반화 범위

- 교육청 본청·교육지원청의 과(課) 단위
- 직속기관(연구정보원, 연수원, 도서관 등) 부서 단위
- 단위학교 교무실 공유폴더

→ 동일한 프로그램이 기관 규모와 무관하게 적용되므로 확산 가능성이 높다.

## 2. 프로그램 개요

### 2.1 한 줄 정의

> 부서 공유폴더를 그대로 둔 채, 내부망 PC 한 대가 문서 본문을 인덱싱하여 "내용으로 찾고, 맥락으로 잇고, 근거와 함께 답하는" 업무 지식베이스

### 2.2 핵심 기능

#### ① 본문 통합검색 (1차 핵심)
- 공유폴더를 주기적으로 스캔하여 **hwp·hwpx·pdf·xlsx·pptx·txt 본문을 텍스트로 추출·인덱싱**
- 키워드 검색 시 파일명이 아니라 **본문이 일치하는 문서**를 하이라이트된 미리보기와 함께 표시
- 결과에서 원본 파일 위치 바로 열기 (파일은 이동·복사하지 않음 — 폴더 체계 그대로 유지)

#### ② 업무 맥락 탐색
- **"작년 이맘때" 보기**: 생산 시기 기준으로 같은 달에 만들어진 문서 묶음 표시 → 연례 반복 업무의 전년도 산출물을 자동으로 보여줌
- **문서 타임라인**: 같은 사업 키워드의 문서를 계획 → 공문 → 결과보고 순으로 시간축에 배열
- **서식 모음**: 표 구조가 동일한 문서(정산서, 명부 등)를 서식 후보로 자동 군집화

#### ③ 디지털 인수인계서 생성
- 담당 업무 키워드를 입력하면 관련 문서를 연도·단계별로 정리한 **인수인계 목차를 자동 생성** → 후임자는 링크 클릭만으로 전임자의 산출물 전체에 접근
- 형식적 인수인계서를 "살아있는 문서 지도"로 대체

#### ④ 근거 기반 질의응답 (2차 확장, 온프레미스 sLLM)
- "이 사업 작년 예산이 얼마였지?", "강사수당 지급 근거 규정이 뭐지?" 같은 자연어 질문에 **근거 문서 링크를 첨부하여** 답변
- 경량 오픈소스 LLM을 내부망 PC(GPU 1장 또는 CPU)에서 구동 — **모든 데이터가 기관 밖으로 한 바이트도 나가지 않음**
- 환각 방지: 근거 문서가 없으면 "관련 문서 없음"으로 응답하는 검색증강(RAG) 구조

### 2.3 기대 효과 (연구 검증 지표)

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 과거 문서 탐색 소요 시간 | 동일 과제 탐색 실험 (전/후) | 15분 → 30초 |
| 문서 재작성(있는데 다시 만든) 건수 | 사용자 설문 + 사례 수집 | 50% 감소 |
| 인수인계 준비 시간 | 인사이동기 실측 | 수일 → 반나절 |
| 신규 전입자 업무 적응 체감 | 5점 척도 설문 | 유의미한 향상 |
| 검색 이용 빈도·성공률 | 시스템 로그 자동 산출 | — |

## 3. 기술 설계

### 3.1 아키텍처 원칙

1. **데이터 외부 유출 제로**: 인덱스·검색·AI 추론 전부 내부망에서 수행. 외부 API 호출 없음
2. **무설치 도입**: 부서의 유휴 PC 1대에 단일 실행파일로 설치, 구성원은 브라우저로 접속
3. **원본 불간섭**: 공유폴더의 파일을 옮기거나 바꾸지 않고 읽기 전용으로 인덱싱 → 도입 거부감 최소화
4. **무료 오픈소스 스택**: 예산 0원으로 어느 기관이든 도입

### 3.2 기술 스택 (제안)

| 계층 | 선택 | 이유 |
|------|------|------|
| 문서 파싱 | hwp/hwpx 파서(오픈소스 hwp5·hwpx는 zip+xml 직접 파싱), pdfminer, openpyxl | 한글 본문 추출이 본 과제의 기술적 핵심 |
| 검색 인덱스 | SQLite FTS5 (1차) → 임베딩 벡터 검색 병행 (2차) | 서버 설치 불필요, 파일 1개로 백업 |
| 백엔드 | Python (FastAPI) | 파서 생태계, 단일 실행파일 패키징(PyInstaller) |
| 프론트 | 서버 렌더링 + 경량 JS | 내부망 저사양 PC·구형 브라우저 대응 |
| sLLM (2차) | 한국어 경량 오픈모델 + llama.cpp 계열 로컬 추론 | 외부 전송 없는 질의응답 |

### 3.3 보안·개인정보 설계 (심사 대응 핵심)

- **접근 제어**: 공유폴더의 기존 권한 체계를 따름 — 읽기 권한이 없는 폴더는 검색 결과에서도 제외
- **민감정보 필터**: 주민등록번호 패턴 등 개인정보가 검출된 문서는 인덱싱 제외 또는 마스킹, 검출 목록을 관리자에게 보고 → **부수 효과로 공유폴더 개인정보 전수점검 기능 수행** (개인정보보호 자체점검과 연계 가능한 어필 포인트)
- **인덱스 암호화 및 접속 로그** 전 구간 기록
- 외부망 차단 환경 동작을 전제로 설계했음을 보고서에 명시

## 5. 연구대회 보고서 구성 포인트

1. **문제의 정량화로 시작**: 소속 기관 구성원 대상 "과거 문서 탐색에 쓰는 시간·재작성 경험" 실태조사 → 체감이 아닌 수치
2. **기술적 연구성**: hwp 본문 추출, 내부망 RAG 등 단순 도구 사용이 아닌 개발 과정의 문제 해결 기록
3. **보안 정면 돌파 서사**: "생성형 AI 시대에 교육기관은 보안 때문에 못 쓴다"는 통념을 **온프레미스 설계로 해결**했다는 점을 차별성의 축으로
4. **이중 효과**: 업무 경감 + 공유폴더 개인정보 전수점검이라는 부수 효과
5. **일반화**: 무료 배포, 설치 매뉴얼, 교육청 과 단위·학교 단위 적용 시나리오 각각 제시

## 6. 리스크와 대응

| 리스크 | 대응 |
|--------|------|
| hwp(구버전 바이너리) 파싱 실패 문서 | hwpx 변환 유도 + 실패 문서는 파일명·메타데이터만 인덱싱 |
| 인덱싱 대상에 비밀·대외주의 문서 포함 | 폴더 단위 제외 설정 + 민감정보 필터 |
| sLLM 답변 품질 | 질의응답은 '근거 문서 제시'를 기본값으로, 생성 답변은 보조로 단계 도입 |
| 유휴 PC 사양 부족 | 검색(1차)은 CPU만으로 동작하도록 설계, sLLM은 선택 기능 |

## 7. 검토 이력

- 1안 "수합·집계 자동화"는 기존 수상작 및 민간 앱(학교종이)과 중복되어 폐기 (2026-06-10)
- 회의록 AI(공공기관용 클로바노트 출시), 상담 챗봇(다수 교육청 운영), AI 문서생성(공공 초거대 AI 실증)도 중복으로 제외
- 차순위 후보였던 "사업 서식 자동 채움 엔진"은 본 시스템의 서식 모음·타임라인 기능과 결합 가능하므로 발전 단계 과제로 보류

# 감사내비 실데이터 구축 가이드 (충북교육청 열린감사방)

연구의 본체인 "지적사례 분석"을 진행하는 절차입니다. 프로그래밍은 필요 없습니다.

## 1. 수집

1. 충북교육청 누리집 → 열린마당 → **열린감사방** → 감사결과 공개 / 감사자료방
2. 최근 3~5년 **종합감사 결과(처분 요구서)** 와 **감사사례집**을 내려받기
   - 대상을 "학교"로 한정 (본청·직속기관 감사는 제외하거나 별도 표기)
3. 파일명을 `연도_기관유형_일련번호` 형식으로 정리 (예: `2024_초_017.hwp`)

## 2. 코딩(분류) — 엑셀 1행 = 지적사항 1건

처분서를 읽으며 지적사항 단위로 엑셀에 입력합니다. 열 순서는
`prototype/tools/엑셀변환.html`의 표와 동일하게:

| 열 | 입력 내용 | 규칙 |
|----|----------|------|
| A 분야 | 분야 id 또는 이름 | gyomu/bokmu/sudang/gyeyak/chehum/hoegye/bangkwa/geupsik/jeongbo/sisul |
| B 제목 | 사례 제목 | 처분서의 "지적 제목"을 유형화해 다듬기 |
| C 내용 | 위반 내용 요약 | **학교명·개인명·금액 등 식별정보 제거**, 유형으로 일반화 |
| D 처분 | 처분 종류 | 처분서 표기 그대로 (주의/경고/시정/회수/징계 등) |
| E 근거 | 근거 규정 | 처분서에 인용된 법령·지침명 (원문 조문 확인 후 기재) |
| F 점검문항 | 자가점검 질문 | "~했는가" 형태로 2~3개, 세미콜론(;) 구분 |
| G 확인문서 | 증빙 문서 | 세미콜론(;) 구분 |
| H 빈출 | O/X | 같은 유형이 3개교 이상에서 반복되면 O |
| I 확인 | O/X | 처분서 원문을 직접 읽고 입력했으면 O |

코딩 요령:
- **같은 유형은 합치고 빈도를 센다.** "출장·초과근무 중복"이 12개교에서 나왔다면 1행으로 만들고 빈출 O. 이 빈도 통계가 보고서의 핵심 근거가 됩니다 (별도 시트에 연도×분야×유형 건수 집계 유지).
- 분류가 애매하면 둘 중 "담당자가 찾을 분야" 기준으로 (예: 방과후 강사료 → 방과후).
- 분야 추가가 필요하면 `data.js`의 CATEGORIES와 변환 도구의 CATEGORIES에 같은 항목을 추가.

## 3. 변환·적용

1. 엑셀 데이터 영역 복사 → `prototype/tools/엑셀변환.html` 열고 붙여넣기 → [변환하기]
2. 결과를 복사해 `prototype/data.js`의 `const CASES = [ ... ];` 전체와 교체
3. `DATA_META`의 `updated`(기준일)와 `status`("2021~2025 종합감사 처분서 N건 분석" 등)를 수정
4. `index.html`을 열어 표시 확인 — 원문 확인(I열 O) 사례는 "⚠ 예시" 배지가 사라짐

## 4. 검증 체크리스트

- [ ] 모든 사례에 기관·개인 식별정보가 없는가
- [ ] 근거 규정을 원문(법제처 국가법령정보센터)에서 확인했는가 (개정·폐지 여부 포함)
- [ ] 빈출(★) 표시가 집계표의 빈도와 일치하는가
- [ ] 감사 부서(감사관실)에 자문·검수를 요청했는가 — 협업 실적으로도 활용

## 5. 갱신 주기

연 1회, 새 감사사례집 발간 직후 위 절차를 반복합니다. 1회차 구축 이후에는
추가분만 입력하면 되므로 연 1~2시간 수준입니다.

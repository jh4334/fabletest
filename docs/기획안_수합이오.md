# 디지털업무개선 연구대회 출품 기획안

## 프로그램명(가칭): 수합이오 — 교내 수합·집계 자동화 플랫폼

---

## 1. 기획 배경

### 1.1 문제 인식

교사의 행정업무 중 가장 반복적이고 시간 소모가 큰 영역은 **"수합(收合)" 업무**다.

- 가정통신문 회신서 수합 (개인정보 동의서, 방과후 수요조사, 현장체험학습 동의서 등)
- 교직원 대상 각종 조사 수합 (연수 신청, 복무 관련 조사, 물품 수요조사 등)
- 담임교사의 학급별 명단 취합 → 업무 담당자의 전교 단위 재취합이라는 **이중 수합 구조**

현재 학교 현장의 일반적인 수합 방식:

1. 업무 담당자가 한글(HWP) 양식을 만들어 메신저로 배포
2. 담임교사가 종이 회신서를 걷거나 학급 단위로 엑셀에 입력
3. 담당자가 학급별 파일 30개를 받아 수작업으로 복사·붙여넣기 집계
4. 미제출자 확인을 위해 메신저로 개별 독촉

→ **단순 조사 1건당 담당자 2~4시간, 담임 30명 × 20분의 노동이 발생**하며, 학기당 수십 건이 반복된다.

### 1.2 기존 도구의 한계

| 도구 | 한계 |
|------|------|
| 구글폼/네이버폼 | 학생·학부모 개인정보가 외부 민간 서버에 저장 → 개인정보보호 지침상 부담 |
| 학교 메신저(쿨메신저 등) | 수합 기능 없음, 파일 단위 주고받기만 가능 |
| NEIS | 정형화된 업무만 지원, 자율 조사 불가 |
| 엑셀 공유 | 동시 편집 충돌, 서식 파괴, 버전 혼란 |

**핵심 차별점: 교내(또는 교육청 내부망) 서버에서 동작하여 개인정보가 외부로 나가지 않는 수합 전용 도구**라는 점이 연구대회에서 설득력 있는 포인트가 된다.

## 2. 프로그램 개요

### 2.1 한 줄 정의

> 업무 담당자가 3분 만에 수합 양식을 만들고, 회신·독촉·집계·문서 출력까지 자동으로 처리하는 학교 전용 웹 플랫폼

### 2.2 핵심 기능

#### ① 양식 빌더 (담당자용)
- 객관식/주관식/파일첨부/서명 항목을 드래그로 조합
- **학교 현장 특화 템플릿 내장**: 개인정보 동의서, 방과후 수요조사, 체험학습 동의서, 급식 알레르기 조사, 연수 신청 등
- 대상 지정: 전교생 / 특정 학년 / 학급 / 교직원 — 학급 명렬표와 자동 연동

#### ② 회신 수집
- 학부모: 가정통신문에 인쇄된 QR코드 또는 문자 링크로 접속, 로그인 없이 학번+이름 확인 후 회신
- 교직원: 내부 계정으로 접속하여 회신
- 종이 회신이 꼭 필요한 경우: 담임이 학급 화면에서 체크박스로 일괄 입력 (하이브리드 모드)

#### ③ 자동 독촉·현황판
- 담당자 화면: 학급별 제출률 실시간 게이지
- 담임 화면: 우리 반 미제출자 명단만 표시
- 마감 전일 미제출자 대상 자동 알림 발송 → **"독촉 메신저 보내기" 업무 소멸**

#### ④ 자동 집계·문서 출력
- 마감 즉시 학급별/학년별/전교 통계 자동 생성 (표·그래프)
- **결재용 산출물 원클릭 출력**: 집계표 엑셀(xlsx), 공문 첨부용 요약 서식(hwpx 호환 양식), 미제출자 명단
- 방과후 수요조사 → 강좌별 신청자 명단 자동 분류 같은 2차 가공 지원

#### ⑤ 학기 단위 재사용
- 작년 조사 양식 복사 → 날짜만 바꿔 재발송
- 학교 간 템플릿 공유(내보내기/가져오기) → **일반화 가능성** 확보

### 2.3 기대 효과 (연구 검증 지표)

| 지표 | 측정 방법 |
|------|----------|
| 조사 1건당 담당자 처리 시간 | 적용 전/후 비교 (목표: 240분 → 30분) |
| 담임교사 학급 수합 시간 | 설문 + 실측 |
| 종이 가정통신문 인쇄량 | 학기 인쇄 매수 비교 (탄소중립 연계 포인트) |
| 회신 회수율·소요일 | 시스템 로그 자동 산출 |
| 사용자 만족도 | 교직원·학부모 설문 (5점 척도) |

## 3. 기술 설계

### 3.1 아키텍처 원칙

1. **개인정보 외부 유출 제로**: 학교 내부 PC 또는 교육청 내부 서버 1대에서 동작. 외부 클라우드·외부 API 미사용
2. **설치 단순성**: 일반화를 위해 "USB에 담아 실행" 수준의 배포 — 단일 실행파일 + SQLite
3. **무료·오픈소스 스택**: 예산 없이 어느 학교든 도입 가능

### 3.2 기술 스택 (제안)

| 계층 | 선택 | 이유 |
|------|------|------|
| 백엔드 | Python (FastAPI) 또는 Node.js | 교사 연구자가 다루기 쉽고 단일 실행파일 패키징 가능 |
| DB | SQLite | 서버 설치 불필요, 파일 1개로 백업 |
| 프론트 | 서버 렌더링 + 경량 JS (또는 React) | 저사양 학교 PC·구형 브라우저 대응 |
| 문서 출력 | openpyxl(엑셀), hwpx 템플릿 치환 | 결재 문화에 필수 |
| 알림 | 학교 문자 발송 시스템 연계용 명단 추출 (1차), API 연동 (2차) | 단계적 구현 |

### 3.3 개인정보 보호 설계 (심사 대응 핵심)

- 학생 식별은 학번+이름 최소 수집, 주민번호류 항목은 양식 빌더에서 원천 차단
- 조사 종료 후 보존기한 경과 시 자동 파기 스케줄러
- 접근 권한 3단계: 관리자 / 업무담당자(본인 조사만) / 담임(본인 학급만)
- 전 구간 접속 로그 기록

## 4. 개발 로드맵

| 단계 | 기간(안) | 산출물 |
|------|---------|--------|
| 1차 MVP | 4주 | 양식 생성 + 링크 회신 + 자동 집계 + 엑셀 출력 |
| 2차 | 4주 | 학급 명렬 연동, 담임 대리입력, 독촉 자동화, 현황판 |
| 3차 | 4주 | hwpx 출력, 템플릿 공유, 자동 파기, 통계 대시보드 |
| 현장 적용 | 1학기 | 소속교 시범 적용 → 효과 측정 → 인근 학교 확산 적용 |

## 5. 연구대회 보고서 구성 포인트

1. **문제의 정량화**: 적용 전 수합 업무 실태조사(소속교 교사 설문 + 업무포털 조사 건수 분석)로 시작 — "체감"이 아닌 수치로 출발
2. **개발 과정의 연구성**: 단순 개발기가 아니라 "교사 요구분석 → 프로토타입 → 사용성 평가 → 개선" 반복 과정을 기록
3. **효과 검증**: 위 2.3 지표의 전/후 비교, 가능하면 2개교 이상 적용
4. **일반화 방안**: 무료 배포, 설치 매뉴얼, 템플릿 공유 체계 — 심사 기준의 '확산 가능성' 직접 대응
5. **차별성 한 줄**: "외부 폼 서비스의 개인정보 문제와 수기 집계의 비효율을 동시에 해결하는 학교 내부형 수합 플랫폼"

## 6. 검토했던 대안 아이디어 (참고)

| 아이디어 | 장점 | 탈락/보류 사유 |
|----------|------|---------------|
| 보결(보강) 자동 배정 시스템 | 교무부 고충 해결, 알고리즘 연구성 높음 | 수혜자가 교무부 소수에 한정, 학교별 규정 편차 큼 |
| AI 생활기록부 문구 보조 | 화제성 높음 | 학생 정보의 외부 AI 전송 문제, NEIS 정책 리스크, 유사 출품 다수 예상 |
| 체험학습 신청·보고서 전자화 | 학부모 체감 큼 | 결재 규정(전자문서 효력) 이슈, 수합 플랫폼의 부분집합으로 흡수 가능 |
| 학교 일정·업무분장 통합 캘린더 | 구현 쉬움 | 기존 도구(구글 캘린더 등) 대비 차별성 약함 |

→ **수합 자동화는 모든 교사가 매주 겪는 보편적 고충이라 효과 측정이 쉽고, 개인정보 보호라는 명확한 차별점이 있어 최우선 추천.**

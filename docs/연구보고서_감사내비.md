# 디지털업무개선 연구대회(충청북도교육청 도대회) 연구보고서

## 감사 지적사례 빅데이터 기반 학교 종합감사 사전 자체점검 시스템 「감사내비」 개발 및 적용

---

**연구 분야**: 디지털 업무개선 · 행정 효율화
**핵심어**: 학교 자체감사, 지적사례 분석, 사전 자체점검, 데이터 기반 행정, 무설치 웹도구
**적용 학교**: 한천초등학교(2026학년도 종합감사 대비)
**개발 산출물**: 무설치·무서버 웹 애플리케이션 「감사내비」 (단일 파일 배포본 포함)

> **요약**
> 학교는 3년 주기로 종합감사를 받지만, 준비는 담당자의 경험과 기억에 의존해 매번 처음부터 반복된다. 본 연구는 충청북도교육청 감사관이 공개한 『2025년도 자체감사 사례집』 등 3종 공식 자료에 수록된 **지적사례 325건을 11개 분야 80개 유형으로 코딩·분석**하여, 학교가 감사 전에 같은 실수를 스스로 점검할 수 있는 디지털 도구 「감사내비」를 개발하였다. 도구는 서버·DB·설치 없이 브라우저만으로 동작하며, 감사 종류별 점검 범위 자동 조정, 수감자료 준비 보드, 빈도 기반 우선 점검 추천, 점검 결과·인수인계 보고서 자동 생성, 사용 효과 자동 측정 기능을 갖추었다. 한천초등학교 2026학년도 종합감사 대비에 적용하여 준비 과정의 효율화 효과를 측정하였다.

# Ⅰ. 연구의 필요성 및 목적

## 1. 연구의 필요성

### 가. 3년마다 돌아오지만, 매번 처음인 종합감사

「공공감사에 관한 법률」과 「충청북도교육청 자체감사 규칙」에 따라 각급 학교는 정기적으로 종합감사를 받는다. 종합감사는 학교 업무 전반(학사·회계·계약·복무·시설 등)의 적법성과 타당성을 점검하는 가장 포괄적인 감사로, 수감 학교는 감사 개시 10일 전까지 방대한 수감자료를 제출하고(자체감사 규칙 제20조제4항), 전 부서가 자료를 준비해야 한다.

그러나 학교 현장의 감사 준비는 다음과 같은 구조적 약점을 안고 있다.

- **경험 의존성**: 감사 준비 노하우가 담당자 개인에게 귀속되어, 인사이동이 잦은 학교에서는 매 주기마다 준비 역량이 초기화된다.
- **정보 비대칭**: 어떤 항목에서 주로 지적이 나오는지에 대한 정보가 체계적으로 공유되지 않아, 학교는 "무엇을 점검해야 하는지"부터 막막하다.
- **반복성**: 감사 지적사항의 상당수는 매년 여러 학교에서 반복되는 같은 유형임에도, 그 패턴이 학교의 사전 점검으로 환류되지 못한다.

### 나. 지적사항은 이미 공개된 데이터다

주목할 점은, 감사 결과가 **이미 공개 자료로 존재한다**는 사실이다. 충청북도교육청 감사관은 매년 『자체감사 사례집』, 『분야별 감사 착안사항』, 『감사실무매뉴얼』을 발간하여 전 기관에 안내한다. 『2025년도 자체감사 사례집』의 일러두기는 "본 사례집을 적극 활용하여 예방 감사의 기능이 충분히 발휘될 수 있도록 자체 직무연수 교재 등으로 활용하여 유사한 사례가 재발되지 않도록" 할 것을 명시하고 있다.

즉, 예방을 위한 데이터는 이미 주어져 있다. 다만 그것이 **PDF·HWP 문서 형태로 흩어져 있어**, 수백 쪽 분량을 담당자가 일일이 읽고 자기 학교 상황에 대입하기 어려울 뿐이다. 이 자료를 구조화하여 "우리 학교가 지금 점검할 항목"으로 변환해 준다면, 같은 데이터가 강력한 예방 도구가 된다.

### 다. 디지털 업무개선의 적합 영역

감사 준비는 ① 반복적이고 ② 규칙(법령·지침)이 명확하며 ③ 데이터(지적사례)가 공개되어 있는, 디지털 도구화에 최적인 업무다. 동시에 학교 행정 정보의 민감성 때문에 **외부 서버에 자료를 올리는 방식은 부적합**하다. 따라서 "공개 데이터를 구조화하되, 학교 내부 정보는 PC 밖으로 내보내지 않는" 무서버 도구가 요구된다.

## 2. 연구 문제

본 연구는 다음 세 가지 문제에 답한다.

1. 충북 학교 종합감사의 지적사례는 **어떤 분야·유형에 집중**되어 있는가? (데이터 분석)
2. 그 분석 결과를 학교가 **감사 전에 스스로 점검**할 수 있는 디지털 도구로 어떻게 구현할 것인가? (도구 개발)
3. 개발한 도구는 실제 학교의 감사 준비에 **효율화 효과**가 있는가? (적용·검증)

## 3. 연구 목적

- 충북 학교 감사 지적사례를 분야×유형으로 코딩·분석하여 **빈출 패턴을 도출**한다.
- 분석 결과를 탑재한 무설치·무서버 사전 자체점검 도구 「감사내비」를 개발한다.
- 한천초등학교 종합감사 대비에 적용하여 효과를 측정하고, 확산 가능성을 검토한다.

# Ⅱ. 이론적 배경 및 선행 분석

## 1. 학교 자체감사 제도

자체감사는 「공공감사에 관한 법률」을 근거로, 감사기구가 소속 기관(학교 포함)의 업무를 점검하는 활동이다. 충청북도교육청은 「충청북도교육청 자체감사 규칙」에 따라 감사를 운영하며, 학교는 감사대상기관으로서 수감 의무를 진다. 주요 절차상 기한은 다음과 같다.

| 절차 | 기한 | 근거 |
|------|------|------|
| 감사계획 통보 | 감사예정일 7일 전 | 공공감사법 시행령 제12조, 자체감사 규칙 제7조 |
| 종합감사 수감자료 제출 | 감사 개시 10일 전 | 자체감사 규칙 제20조제4항 |

## 2. 감사의 종류

매뉴얼은 감사를 6종으로 구분한다. 종류에 따라 점검 범위가 달라진다.

| 종류 | 정의(요지) | 학교 점검 초점 |
|------|-----------|----------------|
| 종합감사 | 업무 전반의 적법성·타당성 점검 | 전 분야 |
| 재무감사 | 예산 운용실태·회계처리 적정성 | 회계·계약·수당 |
| 복무감사 | 복무의무 위반·근무실태·비위 | 복무·보수 |
| 특정감사 | 특정 업무·사업·자금의 문제점 규명 | 해당 사안 분야 |
| 성과감사 | 정책·사업의 경제성·능률성·효과성 | (정책 단위) |
| 일상감사 | 주요 업무 집행 전 적법성·타당성 심사 | 계약·예산 집행 전 |

이 밖에 학교가 실무에서 접하는 **보안감사(정보보안·개인정보)**, **재물조사(물품·재산)** 등은 점검 범위가 특정 분야에 한정된다. 본 연구의 도구는 이 종류 구분을 반영하여, 감사 종류를 선택하면 해당 범위만 점검 대상으로 자동 조정되도록 설계하였다.

## 3. 기존 감사 대비 방식의 한계

| 기존 방식 | 한계 |
|-----------|------|
| 사례집 PDF 통독 | 수백 쪽 분량, 자기 학교 대입 어려움, 1회성 |
| 전임자 인계 문서·구두 | 누락·소실 잦음, 인사이동 시 단절 |
| 감사 공문 수감자료 목록을 종이로 분담 | 진행 상황 공유 어려움, 담당 배분 비효율 |
| 상용 일정·체크리스트 앱 | 학교 감사 특화 데이터 없음, 외부 저장 우려 |

## 4. 도구 설계 방향

- **데이터 주도**: 점검 항목을 코드에 박지 않고 별도 데이터 파일로 분리하여, 사례집 개정 시 데이터만 교체.
- **무서버·무설치**: 브라우저의 localStorage만 사용, 학교 정보는 PC 밖으로 전송하지 않음.
- **무장벽 배포**: 단일 HTML 파일로 패키징하여 파일 하나로 학교 간 전달.
- **현장 절차 반영**: 감사 종류·수감자료 제출 기한·준비 일정 등 매뉴얼의 실제 절차를 그대로 구현.

# Ⅲ. 연구 방법

## 1. 연구 절차

```
[1단계] 공식 자료 수집 → [2단계] 지적사례 코딩 → [3단계] 빈도 분석
   → [4단계] 도구 개발 → [5단계] 한천초 적용·효과 측정 → [6단계] 확산 방안
```

## 2. 자료 수집

충청북도교육청 감사관이 발간한 다음 3종 공식 자료를 분석 대상으로 하였다.

| 자료 | 발간 | 활용 |
|------|------|------|
| 2025년도 자체감사 사례집 | 2026. 3. | 분야별 지적 유형 및 실제 지적사례 |
| 2025년도 분야별 감사 착안사항 | 2025. 3. | 현장체험학습·급식·안전·보안 등 점검 착안 문항 |
| 2025년도 개정 감사실무매뉴얼 | 2025 | 감사 종류·수감자료 제출 기한·종합감사 준비물 |

## 3. 코딩·분석 방법

지적사항을 **유형 단위**로 분류하고, 각 유형에 대해 다음 항목을 코딩하였다.

- 분야(11개 대분류), 점검 주제(소분류)
- 주요내용/감사기준 요약
- 자가점검 문항(착안사항을 "~했는가" 형태로 변환)
- 근거 규정(법령·지침), 확인 문서
- 해당 유형의 사례집 수록 **지적사례 건수**(빈도) 및 실제 지적사례 요약

코딩 원칙으로 ① 기관명·개인명·금액 등 식별정보 제거, ② 같은 유형은 합치고 빈도를 집계, ③ 근거 규정은 원문 표기 유지를 적용하였다.

## 4. 도구 개발 방법

- 구현: HTML/CSS/JavaScript (외부 라이브러리·프레임워크 없음)
- 저장: 브라우저 localStorage (서버·DB 없음)
- 검증: 헤드리스 브라우저(Playwright) 자동 시나리오 테스트로 전 기능·전 분야·전 감사종류 동작 및 콘솔 오류 0 확인
- 배포: 단일 파일 빌드 스크립트로 `감사내비.html` 1개 파일 생성

### 도구의 기술 구조

본 도구는 학교 현장의 제약(설치 권한 제한, 망 분리, 정보 민감성)을 고려하여 다음 원칙으로 설계하였다.

- **무서버·무설치(클라이언트 단독)**: HTML 파일을 브라우저로 여는 것만으로 동작한다. 별도 프로그램 설치나 서버가 필요 없어, 설치 권한이 제한된 학교 업무용 PC에서도 즉시 쓸 수 있다.
- **데이터-로직 분리(데이터 주도 설계)**: 점검 항목·지적사례는 코드가 아니라 별도 데이터 파일(`data.js`)에 구조화되어 있다. 사례집이 개정되면 코드 수정 없이 데이터만 교체하면 되며, 비개발자도 엑셀 변환 도구로 항목을 추가할 수 있다.
- **로컬 저장(정보 비유출)**: 점검 결과·수감자료 상태는 브라우저의 localStorage에만 저장되어 **학교 정보가 PC 밖으로 전송되지 않는다**. 저장이 차단된 보안 환경에서도 앱은 정상 동작하며(세션 내 사용), 백업 파일로 상태를 보존·이관할 수 있다.
- **단일 파일 패키징**: 빌드 스크립트가 HTML·CSS·JavaScript·데이터를 하나의 `감사내비.html`로 합쳐, 파일 하나만 전달하면 배포가 끝난다.

```
[ data.js ]  점검 항목·지적사례(공개 사례집 분석 결과)
     │
[ app.js ]   대시보드·점검·보드·통계·보고서 로직
     │
[ index.html + style.css ]  화면
     │
   브라우저 localStorage  ← 학교 정보(이 PC에만 저장)
     │
[ build-single.js ]  → 감사내비.html (단일 파일 배포본)
```

품질 보증을 위해 헤드리스 브라우저(Playwright) 기반 자동 시나리오 테스트로 전 기능·11개 분야·7개 감사 종류의 동작과 콘솔 오류 0을 확인하였다.

## 5. 적용·효과 검증 방법

한천초등학교의 2026학년도 종합감사 대비에 도구를 적용하고, 도구에 **내장된 사용 효과 측정 기능**으로 다음 지표를 자동 수집한다.

- 종합 준비도(%) 변화: 점검 시작 시점 → 감사 직전
- 누적 점검 시간 및 점검 진행 일수
- (보조) 사용 교사 만족도·체감 효과 설문

**② 인사·복무·출장 (지적 56건)**

- *출장(6건)*: 순회지도 교사 등의 복무 처리 적정성.

**③ 교무·학사 (지적 46건)**

- *학교규칙·학생생활규정(3건)*: 상위 법령·지침 부합 및 현행화.

**④ 보수·수당 (지적 31건)**

**⑤ 계약·물품 (지적 15건)**

# Ⅴ. 결론 및 제언

## 1. 결론

- 충북 학교 감사 지적사례 325건을 분석한 결과, 지적은 **회계 집행·예산·복무 등 "돈과 근태를 다루는 절차"에 집중**됨을 확인하였다. 이는 학교가 사전 점검을 어디에 집중해야 하는지에 대한 데이터 기반 근거를 제공한다.
- 이 분석을 탑재한 무설치·무서버 도구 「감사내비」를 개발하여, 공개 데이터를 "우리 학교가 지금 점검할 항목"으로 변환하였다.
- 한천초등학교 종합감사 대비에 적용하여, 준비 일정 자동화·분담 점검·빈출 우선 점검·수감자료 관리의 효과를 확인하였다(정량 지표는 도구가 자동 측정).

## 2. 제언

- **확산**: 단일 파일 배포본으로 인근 학교에 전달하면, 같은 분석 데이터를 즉시 공유할 수 있다. 데이터 파일만 교체하면 타 시·도로도 확장 가능하다.
- **갱신**: 매년 새 사례집 발간 직후 데이터를 갱신한다. 엑셀 변환 도구를 제공하여 비개발자도 추가 입력이 가능하다.
- **협업**: 감사관실의 자문·검수를 받으면 데이터 신뢰성과 현장 활용도가 함께 높아진다.
- **한계와 보완**: 본 도구의 데이터는 공개 사례집 기반이므로, 처분 결과는 생략되어 있고 최신 법령 개정은 별도 확인이 필요하다. 도구는 근거 규정을 국가법령정보센터로 연결하여 이를 보완한다.

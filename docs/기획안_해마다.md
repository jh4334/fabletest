# 디지털업무개선 연구대회 출품 기획안

## 프로그램명(가칭): 해마다 — 연간 반복업무 내비게이션

---

## 1. 기획 배경

### 1.1 문제 인식: 교육기관 업무의 80%는 "작년에도 했던 일"이다

교육청·직속기관·학교의 업무는 대부분 연간 주기로 반복된다. 그런데 그 반복 업무가 매년 처음 하는 일처럼 처리된다.

- 담당자가 바뀌면 "이 업무를 언제, 어떤 순서로, 어떤 서식으로 하는지"가 통째로 사라짐 (인수인계서는 형식적 1~2장)
- 시기를 놓쳐서 생기는 업무 사고: 보고 기한 누락, 공모 신청 시기 놓침, 정산 지연
- 전임자에게 전화로 묻거나, 공유폴더를 뒤지거나, 옆 기관에 물어보는 데 드는 시간
- 업무분장표(엑셀)는 "누가 무엇을"만 있고 **"언제, 어떻게"가 없음**

### 1.2 기존 도구로 안 되는 이유

| 도구 | 한계 |
|------|------|
| 업무분장표(엑셀) | 정적 문서. 시기·절차·자료 위치 없음, 알림 없음 |
| 업무관리시스템 일정 | 개인 일정 수준. 업무 노하우·이력 축적 구조 없음 |
| 노션·트렐로 등 협업툴 | 행정자료의 외부 클라우드 저장 불가(보안 지침) |
| 개인 수첩·한글 메모 | 담당자가 떠나면 함께 사라짐 |

**공백 지점: 내부망에서 쓸 수 있는, "기관의 연간 업무 그 자체"를 자산으로 축적하는 도구가 없다.**

### 1.3 기술적 포지션 (의도적 단순화)

본 연구는 AI·문서파싱 같은 고난도 기술을 쓰지 않는다. 핵심 가치는 기술이 아니라 **"업무 지식을 카드 한 장으로 구조화하는 틀"**과 그것이 해마다 자동으로 되살아나는 순환 구조에 있다. 단순한 기술로 만든 도구가 현장에 정착하기 쉽다는 점 자체를 연구의 논지로 삼는다.

## 2. 프로그램 개요

### 2.1 한 줄 정의

> 부서의 모든 반복 업무를 "업무 카드"로 등록해두면, 매년 제때 담당자에게 다시 꺼내주고, 할수록 노하우가 쌓이는 내부망 업무 달력

### 2.2 핵심 개념: 업무 카드

업무 1건 = 카드 1장. 카드에 담기는 것:

| 항목 | 예시 |
|------|------|
| 업무명 | 정보화기기 임차 계약 갱신 |
| 시기 | 매년 2월 2주 시작, 3월 말 마감 |
| 담당 | ○○과 ○○○ |
| 절차 체크리스트 | ① 수요조사 → ② 품의 → ③ 계약 의뢰 → ④ 검수 |
| 자료 위치 | 공유폴더 경로·작년 산출물 링크 (파일을 복사하지 않고 경로만 기록) |
| 노하우 메모 | "계약부서 협조는 최소 3주 전에", "작년에 ○○ 누락해서 반려됨" |
| 연도별 이력 | 2025년 완료(메모), 2026년 진행 중 |

### 2.3 핵심 기능

#### ① 이번 달 대시보드 (1차 핵심)
- 접속하면 "이번 달 시작할 업무 / 마감 임박(D-7) / 지연" 카드가 자동 표시
- 부서 전체 보기와 내 업무 보기 전환

#### ② 연간 업무 달력
- 부서의 1년 업무가 월별로 한눈에 보이는 보드 → 업무분장 회의·신학년 준비의 기초자료
- 카드를 끌어서 시기 조정

#### ③ 자동 순환 (이 도구의 심장)
- 카드를 "올해 완료" 처리하면 **내년도 카드가 자동 생성**되고, 올해의 메모·산출물 링크가 이력으로 축적
- 3년만 쓰면 카드 하나에 3개년 노하우가 쌓임 → 시간이 지날수록 가치가 커지는 구조

#### ④ 원클릭 인수인계
- 담당자를 선택하면 그 사람의 업무 카드 전체(시기·절차·자료 위치·이력 포함)가 **인쇄용 인수인계서로 자동 출력**
- 인사이동 시 카드의 담당자만 바꾸면 인수인계 완료

#### ⑤ 아침 알림 (선택 기능)
- 시작·마감 임박 업무를 정리한 알림 텍스트 자동 생성 → 부서 메신저에 붙여넣기 (시스템 연동 없이 복사 방식 → 구현 부담 제로)

### 2.4 기대 효과 (연구 검증 지표)

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 업무 시기 누락·지연 건수 | 적용 전/후 부서 사례 집계 | 80% 감소 |
| 인수인계서 작성 시간 | 인사이동기 실측 | 수일 → 클릭 1회 |
| 전임자 문의(전화·메신저) 빈도 | 신규 전입자 설문 | 50% 감소 |
| 작년 자료 찾는 시간 | 카드 링크 경유 vs 폴더 탐색 비교 | 분 단위 → 초 단위 |
| 사용 정착도 | 카드 등록 수·접속 로그 | — |

## 3. 기술 설계 (단순함이 전략)

### 3.1 원칙

1. **외부 유출 제로**: 내부망 PC 1대에서 동작, 외부 API·클라우드 없음
2. **파싱·AI 없음**: 파일은 열지 않고 경로만 기록 → 기술 리스크 제거
3. **무설치 도입**: 단일 실행파일 + 데이터는 SQLite 파일 1개 (백업 = 파일 복사)
4. **2주 안에 MVP**: 교사 1인이 만들 수 있는 규모

### 3.2 스택

| 계층 | 선택 | 이유 |
|------|------|------|
| 백엔드 | Python Flask (또는 FastAPI) + SQLite | 가장 단순한 검증된 조합 |
| 프론트 | 서버 렌더링(Jinja) + 최소한의 JS | 내부망 구형 브라우저 대응, 빌드 도구 불필요 |
| 인쇄 출력 | 브라우저 인쇄용 CSS | hwp 생성 불필요 — 인쇄·PDF 저장으로 충분 |
| 배포 | PyInstaller 단일 exe | USB로 옮겨 더블클릭 설치 |

데이터 모델은 사실상 테이블 3개: 업무카드, 연도별이력, 사용자. 이 단순함이 유지보수·일반화의 무기다.

### 3.3 보안

- 내부망 전용, 학생·학부모 개인정보를 다루지 않음(업무 메타정보만) → 개인정보 리스크 원천 차단
- 부서 단위 설치라 권한 체계도 단순 (관리자/구성원 2단계)

## 4. 개발 로드맵

| 단계 | 기간(안) | 산출물 |
|------|---------|--------|
| 0차 검증 | 1주 | 공유 엑셀로 업무 카드 양식 프로토타입 → 부서 동료 피드백으로 카드 항목 확정 |
| 1차 MVP | 2주 | 카드 등록·수정 + 이번 달 대시보드 + 연간 달력 |
| 2차 | 2주 | 완료→내년 카드 자동 생성, 연도별 이력, 인수인계 인쇄 |
| 3차 | 2주 | 알림 텍스트, 통계(누락·지연 현황), exe 패키징·매뉴얼 |
| 현장 적용 | 1학기 | 소속 부서 적용 → 효과 측정 → 타 부서·학교 1곳 확산 적용 |

0차 검증 단계가 중요하다: 코드 한 줄 쓰기 전에 엑셀 프로토타입으로 카드 구조의 현장성을 확인하면, 연구보고서의 "요구분석 → 설계 → 개선" 서사도 자연스럽게 만들어진다.

## 5. 연구대회 보고서 구성 포인트

1. **문제의 정량화**: 부서 구성원 대상 "업무 시기 놓친 경험, 전임자 문의 빈도, 인수인계 소요 시간" 실태조사로 시작
2. **'단순함'을 논지로**: 고도 기술 없이도 업무 구조화만으로 경감 효과를 낸다는 역발상 — AI 일색일 출품작들 사이에서 오히려 차별화
3. **시간이 지날수록 가치가 커지는 구조**: 1년차보다 3년차에 더 유용함을 이력 축적 구조로 논증 → 지속가능성 항목 직접 대응
4. **이중 효과**: 업무 경감 + 조직 지식 자산화(인사이동에 강한 조직)
5. **일반화**: 교육청 과 단위 / 직속기관 부서 / 학교 교무실 적용 시나리오 각각 제시, 무료 배포·설치 매뉴얼 첨부

## 6. 리스크와 대응

| 리스크 | 대응 |
|--------|------|
| 초기 카드 입력 부담 ("또 입력거리냐") | 업무분장표 엑셀 가져오기 기능 + 카드 필수항목 3개로 최소화, 나머지는 점진 입력 |
| 한 번 쓰고 방치 | 대시보드를 부서 회의 자료로 활용하는 운영 루틴 제안 (도구+운영 모델 세트로 보고) |
| "그냥 엑셀로 되지 않냐"는 심사 반론 | 자동 순환·알림·인수인계 출력은 엑셀로 불가함을 비교표로 제시. 0차 엑셀 프로토타입의 한계 기록이 곧 근거가 됨 |
| 부서마다 업무 체계 상이 | 카드 구조를 고정하되 분류 라벨은 자유 설정 |

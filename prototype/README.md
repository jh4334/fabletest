# 감사내비 프로토타입 (충북 · 학교용)

서버·DB·설치 프로그램 없이 동작하는 학교 종합감사 사전 자체점검 도구입니다.

## 실행 방법

`index.html`을 더블클릭해서 브라우저(엣지/크롬)로 열면 끝입니다.
학교 공유폴더에 `prototype` 폴더째 복사해두면 전 교직원이 사용할 수 있습니다.
점검 체크·수감자료 상태는 각자 PC의 브라우저(localStorage)에만 저장되며 외부로 전송되지 않습니다.

## 화면 구성

| 탭 | 기능 |
|----|------|
| 대시보드 | 감사 D-day, 종합 준비도(%), 분야별 점검 현황(미진 분야 우선), 빈출(★) 사례 추천 점검 |
| 분야별 자체점검 | 10개 분야(교무·학사/복무/수당/계약/체험학습/회계/방과후/급식/개인정보/시설)의 지적사례 + 근거 규정 + 자가점검 체크리스트. 키워드 검색, 사례별 메모(인수인계용), '해당없음' 처리 지원 |
| 수감자료 보드 | 감사 공문의 수감자료 목록을 엑셀에서 복사→붙여넣기로 가져와 담당 배분·준비 상태 관리, 준비율 표시, CSV 내보내기. [예시 목록으로 체험] 버튼으로 바로 시험 가능 |
| 통계·보고서 | 분야별 사례 분포·처분 유형 차트, 점검 진행률, **결재란이 포함된 점검 결과 보고서 인쇄/PDF**, 점검 CSV, **JSON 백업/복원**(다른 PC·후임자에게 상태 이관) |

브라우저 보안 설정으로 저장이 차단된 PC에서도 앱은 정상 동작하며(세션 내 사용),
이 경우 화면 하단에 안내가 표시되고 백업 파일 저장으로 상태를 보존할 수 있습니다.

## 수록 데이터 (충북교육청 감사관 공식 자료 기반)

`data.js`의 사례·점검문항은 충청북도교육청 감사관이 발간한 다음 공식 자료를 분석해 구축했습니다.

| 출처 | 반영 내용 |
|------|-----------|
| 2025년도 자체감사 사례집 (2026. 3.) | 학사·재무회계·사학·인사복무·행정일반·범죄/갑질 분야 지적 유형 및 실제 지적사례 |
| 2025년도 분야별 감사 착안사항 (2025. 3.) | 현장체험학습·급식·안전·보안 등 보강 분야 점검 착안 문항 |
| 2025년도 개정 감사실무매뉴얼 | 수감자료 제출 기한(개시 10일 전), 종합감사 기본 준비물 목록 |

- **11개 분야 · 80개 점검 주제 · 점검문항 415개 · 실제 지적사례 65개 주제 325건** 수록
- 사례집 방침에 따라 **처분 결과는 생략**되어 있어, 통계는 "분야별 지적 빈도(건수)"로 표시됩니다
- `freq`(사례집 수록 지적사례 건수)가 3건 이상이면 **★ 빈출**로 우선 점검 추천에 노출됩니다
- 사립학교·법인 분야는 **사립 기관만 해당** — 공립학교는 분야 상단의 *[이 분야 전체 해당없음]* 한 번으로 제외

### 데이터 갱신·보강

새 사례집이 나오면 갱신합니다. 직접 JS를 고치지 않아도, 엑셀로 사례를 코딩한 뒤
**`tools/엑셀변환.html`** 에 붙여넣으면 CASES 코드가 자동 생성됩니다.
수집·코딩 절차는 `docs/데이터수집_가이드.md` 참고.

### data.js 입력 형식

```js
{
  id: "분야-번호",            // 예: "bokmu-04"
  cat: "bokmu",              // CATEGORIES의 id 중 하나
  verified: true,            // 원문 확인 여부
  freq: 3,                   // 사례집 수록 지적사례 건수(3 이상이면 ★ 빈출)
  title: "점검 주제 제목",
  detail: "주요내용/감사기준 요약",
  disposition: "",           // 사례집은 처분 생략 → 보통 빈 문자열
  basis: "근거 법령·지침",
  checks: ["착안 점검 문항 1", "문항 2"],
  docs: ["확인할 문서 1", "문서 2"],
  examples: ["실제 지적사례 요약 1", "요약 2"]   // 선택 — 카드에서 접기/펼치기로 표시
}
```

분야를 추가하려면 `CATEGORIES` 배열에 `{ id, name, desc }`를 추가하면 됩니다.
코드(app.js) 수정은 필요 없습니다.

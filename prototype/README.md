# 감사내비 프로토타입 (충북 · 학교용)

서버·DB·설치 프로그램 없이 동작하는 학교 종합감사 사전 자체점검 도구입니다.

## 실행 방법

`index.html`을 더블클릭해서 브라우저(엣지/크롬)로 열면 끝입니다.
학교 공유폴더에 `prototype` 폴더째 복사해두면 전 교직원이 사용할 수 있습니다.
점검 체크·수감자료 상태는 각자 PC의 브라우저(localStorage)에만 저장되며 외부로 전송되지 않습니다.

## 화면 구성

| 탭 | 기능 |
|----|------|
| 대시보드 | 감사 D-day, 종합 준비도(%), 분야별 점검 현황(미진 분야 우선), 빈출(★) 사례 추천 점검 |
| 분야별 자체점검 | 10개 분야(교무·학사/복무/수당/계약/체험학습/회계/방과후/급식/개인정보/시설)의 지적사례 + 근거 규정 + 자가점검 체크리스트. 키워드 검색, 사례별 메모(인수인계용), '해당없음' 처리 지원 |
| 수감자료 보드 | 감사 공문의 수감자료 목록을 엑셀에서 복사→붙여넣기로 가져와 담당 배분·준비 상태 관리, 준비율 표시, CSV 내보내기. [예시 목록으로 체험] 버튼으로 바로 시험 가능 |
| 통계·보고서 | 분야별 사례 분포·처분 유형 차트, 점검 진행률, **결재란이 포함된 점검 결과 보고서 인쇄/PDF**, 점검 CSV, **JSON 백업/복원**(다른 PC·후임자에게 상태 이관) |

브라우저 보안 설정으로 저장이 차단된 PC에서도 앱은 정상 동작하며(세션 내 사용),
이 경우 화면 하단에 안내가 표시되고 백업 파일 저장으로 상태를 보존할 수 있습니다.

## ⚠ 데이터 교체가 본 연구의 핵심 작업입니다

`data.js`에 수록된 사례는 학교 종합감사의 **대표 지적 유형으로 구성한 예시 시드 데이터**입니다
(각 사례에 `verified: false` 표시). 실제 활용·연구를 위해서는:

1. 충북교육청 누리집 → 열린마당 → **열린감사방**의 감사결과 처분서·감사사례집 수집 (최근 3~5년)
2. 사례를 읽고 분야 × 유형으로 분류(코딩)
3. `data.js`의 `CASES` 배열을 실제 사례로 교체·보강
   - 기관·개인 식별정보는 제거하고 유형화하여 기재
   - 근거 규정은 원문 조문 확인 후 기재, 확인 완료 시 `verified: true`로 변경
4. `DATA_META.updated` 갱신

### data.js 입력 형식

```js
{
  id: "분야-번호",            // 예: "bokmu-04"
  cat: "bokmu",              // CATEGORIES의 id 중 하나
  verified: true,            // 처분서 원문 확인 여부
  title: "지적사례 제목",
  detail: "유형화한 위반 내용 요약",
  disposition: "주의",        // 처분 종류 (주의/경고/시정/회수 등)
  basis: "근거 법령·지침",
  checks: ["자가점검 문항 1", "문항 2"],
  docs: ["확인할 문서 1", "문서 2"]
}
```

분야를 추가하려면 `CATEGORIES` 배열에 `{ id, name, desc }`를 추가하면 됩니다.
코드(app.js) 수정은 필요 없습니다.
